# Инструменты моделирования инженерных сетей

Набор из пяти инструментов для работы с MEP-элементами в Revit. Расположены в двух стеках панели **Общее**: **Моделирование** и **Моделирование 2**.

---

## 1. Вращать MEP

**Папка:** `ВращатьЭлементы.pushbutton`

Вращает выбранные MEP-элементы вокруг оси, заданной коннектором другого элемента. Поддерживает интерактивный предпросмотр.

**Порядок работы:**

1. Выберите элемент, который нужно вращать (или выделите заранее).
2. Выберите элемент рядом с нужным коннектором — его коннектор определит точку и ось вращения.
3. В открывшемся окне настройте параметры:
   - **Угол** — по умолчанию 90°, шаг изменения ±15°.
   - **По часовой / против часовой** — кнопки мгновенного поворота с предпросмотром.
   - **Вращать подключённые элементы** — включает обход графа соединений; вся присоединённая ветка поворачивается вместе с выбранным элементом. Элементы, подключённые вдоль оси вращения, исключаются.
   - **Включая продольно подключённые** — добавляет к набору вращения элементы, чьи коннекторы направлены вдоль оси.
4. Нажмите **Принять** для подтверждения или **Отмена** для отката всех изменений.

**Что поддерживается:** трубы, воздуховоды, лотки, фитинги и другие MEP-элементы.

## 2. Выровнять по высоте

**Папка:** `Выровнять по высоте.pushbutton`

Выравнивает одну MEP-линию по высоте относительно другой в точке их пересечения или проекции.

**Порядок работы:**

1. Выберите **базовую** MEP-линию (трубу, воздуховод, лоток и т.д.), по высоте которой нужно выровнять.
2. Выберите **перемещаемую** MEP-линию, которую нужно поднять или опустить.

Скрипт определяет геометрическую ситуацию:

- **Пересекающиеся в плане** — вычисляется отметка каждой линии в точке пересечения осей, перемещаемая линия сдвигается по Z до совпадения.
- **Соосные (продолжение)** — находится отметка в стыке, вторая линия поднимается/опускается.
- **Параллельные со смещением** — проекция ближайшего конца перемещаемой линии на базовую, выравнивание по Z в этой точке.

Уклон обеих линий сохраняется — выполняется только вертикальный сдвиг.

## 3. Соединить коннекторы

**Папка:** `СоединитьКоннекторы.pushbutton`

Соединяет два MEP-элемента через ближайшие свободные коннекторы. Автоматически объединяет соосные трубы/воздуховоды в один элемент.

**Порядок работы:**

1. Выберите два элемента с MEP-коннекторами (можно выделить заранее или указать кликами).
2. Скрипт находит пару ближайших **свободных** коннекторов (по одному от каждого элемента).
3. Выполняется:
   - Второй элемент **перемещается** к первому так, чтобы коннекторы совпали.
   - Если элементы — соосные MEP-линии (два отрезка трубы на одной оси), они **объединяются** в один элемент.
   - В остальных случаях коннекторы **соединяются** через `ConnectTo`; при неудаче создаётся **муфта** (`NewUnionFitting`).

## 4. Разделить

**Папка:** `Разделить.pushbutton` *(стек Моделирование 2)*

Нарезает выбранные лотки на отрезки заданной длины.

**Порядок работы:**

1. Выберите длину сегмента: **2000**, **3000** или **6000 мм**.
2. Выберите один или несколько лотков.
3. Каждый лоток заменяется набором новых сегментов указанной длины с сохранением типа, уровня, высоты, ширины и смещения.

Лотки короче выбранной длины сегмента пропускаются.

---

## 5. Спринклер

**Папка:** `Спринклер.pushbutton` *(стек Моделирование 2)*

Подключает спринклер к трубе с автоматическим созданием врезки.

**Порядок работы:**

1. Выберите **трубу**, к которой нужно подключиться.
2. Выберите **спринклер**.

Скрипт автоматически:

- Находит точку врезки — проекцию коннектора спринклера на ось трубы.
- Проверяет, что спринклер не подключён и не слишком близко к трубе.
- Создаёт **врезку** (семейство `ADSK_СтальСварка_Врезка`) на трубе.
- Перемещает спринклер к открытому коннектору врезки и соединяет его.

**Ограничения:** работает только со спринклерами и трубами; врезка должна быть загружена в проект.

---

## Общие рекомендации

- Все инструменты поддерживают отмену (Ctrl+Z) в Revit.
- Перед массовым использованием рекомендуется проверить результат на одном элементе.
- Инструменты корректно обрабатывают ошибки — при неудаче изменения откатываются, пользователь получает уведомление.
