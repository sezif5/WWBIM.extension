# ДГП Параметры

## Описание

Скрипт заполняет параметры экземпляров ДГП (ДГП = Дополнительные Графические Параметры) в модели Revit на основе правил сопоставления по категории и условию.

## Принцип работы

1. Скрипт перебирает все категории, для которых заданы правила
2. Для каждого элемента проверяется:
   - Категория элемента
   - Значение условия (параметр "Группа модели" в типе или имя схемы зонирования для Areas)
3. При совпадении условия выполняется назначение значений из исходных параметров в целевые
4. Формируется отчет о выполнении

## Условия сопоставления

### Для большинства категорий
Условие берется из параметра **"Группа модели"** в типе элемента.

Пример: Окно заполнится значениями высоты и ширины, если параметр "Группа модели" в типе окна равен `"Заполнение оконных проёмов"`.

### Для категории Зоны (OST_Areas)
Условие берется из **имени схемы зонирования**.

Примеры:
- Схема `"СПП в ГНС"` → Код="9999", Наименование="9999", Площадь из параметра "Площадь"
- Схема `"Общая площадь"` → Код="П3 03", Наименование="9999", Площадь из параметра "Площадь"

## Параметры

### Целевые параметры (ДГП)
- `ДГП_Площадь земельного участка`
- `ДГП_Площадь`
- `ДГП_Длина`
- `ДГП_Толщина`
- `ДГП_Высота`
- `ДГП_Ширина`
- `ДГП_Диаметр`
- `ДГП_Код помещения и зоны МССК`
- `ДГП_Наименование помещения и зоны МССК`
- `ДГП_Наименование проекта`
- `ДГП_Адрес`

### Исходные параметры
- `Площадь`
- `ADSK_Площадь с коэффициентом`
- `ADSK_Размер_Высота`
- `ADSK_Размер_Ширина`
- `ADSK_Размер_Диаметр`
- `Толщина`
- `Наименование проекта`
- `Адрес проекта`

### Специальные значения
- `const:XXX` — константа (прямое присваивание значения)
- `type:XXX` — параметр берется из типа элемента (а не экземпляра)

## Отчет

После выполнения скрипт выводит:

### Общая статистика
- Обработано элементов
- Обновлено
- Пропущено

### По категориям
Для каждой категории: обработано, обновлено, пропущено

### Причины пропусков
- `parameter_not_found` — целевой параметр не найден
- `wrong_storage_type` — несовпадение типа параметра
- `readonly` — параметр только для чтения
- `already_ok` — значение уже соответствует ожидаемому
- `source_empty` — исходное значение пусто
- `exception` — ошибка при записи
- `no_rule` — не найдено правило для элемента

### Примеры no_rule (до 60)
Элементы, для которых не найдено подходящее правило, с указанием:
- ID элемента
- Категория
- Фактическое значение "Группа модели"
- Ожидаемые значения (какие группы принимаются правилом)

## Использование

1. Запустить инструмент из Revit
2. Дождаться завершения и изучить отчет

## Примечания

- Параметр "Группа модели" проверяется **только в типе** элемента (не в экземпляре)
- Для категорий с правилами без условия (default) правило применяется ко всем элементам этой категории
- Если для элемента не найдено правило, он пропускается и добавляется в список `no_rule_samples`
- Значения нормализуются перед сравнением (удаляются лишние пробелы, неразрывные пробелы заменяются на обычные)
