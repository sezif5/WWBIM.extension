# ДГП Параметры

## Описание

Скрипт заполняет параметры экземпляров ДГП (ДГП = Дополнительные Графические Параметры) в модели Revit на основе правил сопоставления по категории и условию.

## Принцип работы

1. Скрипт перебирает все категории, для которых заданы правила
2. Для каждого элемента проверяется:
   - Категория элемента
   - Значение условия (параметр "Группа модели" в типе или имя схемы зонирования для Areas)
3. При совпадении условия выполняется назначение значений из исходных параметров в целевые
4. Формируется отчет о выполнении

## Условия сопоставления

### Для большинства категорий
Условие берется из параметра **"Группа модели"** в типе элемента.

Пример: Окно заполнится значениями высоты и ширины, если параметр "Группа модели" в типе окна равен `"Заполнение оконных проёмов"`.

### Для категории Зоны (OST_Areas)
Условие берется из **имени схемы зонирования**.

Примеры:
- Схема `"СПП в ГНС"` → Код="9999", Наименование="9999", Площадь из параметра "Площадь"
- Схема `"Общая площадь"` → Код="П3 03", Наименование="9999", Площадь из параметра "Площадь"

### Специальные значения source
- `const:XXX` — константа (прямое присваивание значения)
- `type:XXX` — параметр берется из типа элемента (а не экземпляра)
- `cond:Параметр=Значение?ЕслиДа:ЕслиНет` — условное заполнение. Если параметр пустой — элемент отмечается как ошибка (источник пуст). Пример: `cond:Назначение=Квартира?Жилая зона:Нежилая зона`

## Отчет

После выполнения скрипт выводит:

### Общая статистика
- Всего элементов
- Заполнено (записано новых значений)
- Уже заполнено (значения были корректны, изменений не потребовалось)
- С проблемами (реальные ошибки: параметр не найден, источник пуст и т.д.)

### По категориям
Для каждой категории: обработано, заполнено, уже заполнено, с проблемами

### Причины проблем
- Не найден целевой параметр
- Неверный тип целевого параметра
- Целевой параметр только для чтения
- Пустое исходное значение (включая числовое значение 0)
- Ошибка при записи значения
- Не найдено подходящее правило
- Отсутствует параметр ADSK_Размер_

### Проблемные элементы
Таблица с дедупликацией по типу элемента — для каждого типа показывается только один экземпляр с конкретной проблемой. Все ID кликабельные.

Столбцы:
- **ID элемента** — кликабельная ссылка
- **Категория** — русское название
- **Ситуация** — краткое описание проблемы
- **Подробности** — человекочитаемое описание: какой параметр пуст/отсутствует/заблокирован и что нужно сделать

## Использование

1. Запустить инструмент из Revit
2. Дождаться завершения и изучить отчет

## Примечания

- Параметр "Группа модели" проверяется **только в типе** элемента (не в экземпляре)
- Для категорий с правилами без условия (default) правило применяется ко всем элементам этой категории
- Если для элемента не найдено правило, он пропускается и добавляется в список проблем
- Значения нормализуются перед сравнением (удаляются лишние пробелы, неразрывные пробелы заменяются на обычные)
- Числовое значение 0 считается пустым источником и не записывается в целевой параметр
- Для стен с группой модели "Заполнение витражных проёмов" (не "оконных") заполняются высота и ширина
